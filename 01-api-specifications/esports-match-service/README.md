# 🎮 Esports Match Service API

> Спецификация REST API для сервиса управления киберспортивными матчами

## 📋 О проекте

**Тип работы**: Учебный проект в рамках обучения на системного аналитика  
**Цель**: Разработать спецификацию API для сервиса работы с матчами и игровыми событиями в сфере киберспорта.

### 🎯 Бизнес-требования
- Сервис предоставляет возможность работы с матчами
- Разделение функционала: **Клиент** (только просмотр) и **Админ** (полный доступ)
- Поддержка трёх игр на старте: `Dota 2`, `CS 2`, `Overwatch 2`
- Учёт статусов матча и событий: `не началось` / `в процессе` / `закончено`

---

## 🔧 Технологии

| Категория | Инструмент |
|-----------|------------|
| **Спецификация** | OpenAPI 3.0.3 |
| **Формат** | YAML |
| **Архитектура** | REST |
| **Аутентификация** | ApiKey (header) |
| **Идемпотентность** | `Idempotency-Key` header |

---

## 📦 Функционал API

### 🔓 Публичные методы (доступны всем)

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/matches` | Получить список матчей с пагинацией и фильтрацией |
| `GET` | `/matches/{id}` | Получить детали матча + вложенные события |

**Параметры фильтрации для `/matches`:**
```yaml
page: integer      # Номер страницы (по умолчанию: 1)
size: integer      # Записей на странице (1-100, по умолчанию: 20)
game: enum         # Фильтр по игре: Dota 2, CS 2, Overwatch 2
search: string     # Текстовый поиск по названию матча или команде
status: enum       # Фильтр по статусу: NOT_STARTED, IN_PROGRESS, FINISHED
