# 💸 Transfer Coordination Service API

> Спецификация REST API для сервиса координации переводов между банковскими счетами

## О проекте

**Тип работы**: Учебный проект в рамках обучения на системного аналитика  
**Цель**: Разработать спецификацию API для сервиса создания и координации денежных переводов между счетами.

### Бизнес-требования
- Сервис предоставляет возможность создания переводов между счетами
- Инициализация транзакции в смежных системах
- Возврат актуального баланса счёта отправителя после операции
- Поддержка идемпотентности запросов для предотвращения дублей
- Валидация валюты по стандарту ISO 4217

---

## Технологии

| Категория | Инструмент |
|-----------|------------|
| **Спецификация** | OpenAPI 3.0.3 |
| **Формат** | YAML |
| **Архитектура** | REST |
| **Аутентификация** | ApiKey (заголовок `X-API-Key`) |
| **Идемпотентность** | Заголовок `transferId` (UUID v4) |
| **Контур** | Prod + Test |

## Функционал API

### Публичные методы 

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/transfers` | Создать новый перевод между счетами |

**Параметры запроса для `/transfers` (POST):**

```yaml
transferId: uuid       # Уникальный ключ идемпотентности (в заголовке)
senderClientId: string # Идентификатор клиента-отправителя
withdrawAmount: object # Сумма и валюта перевода
  value: number        # Сумма (минимум: 0.01, точность: 2 знака)
  currency: string     # Код валюты ISO 4217 (RUB, USD, EUR)
withdrawAccount: object # Счёт списания
  accountId: string    # Номер счёта (20 цифр)
  currency: string     # Валюта счёта
depositAccount: object  # Счёт зачисления
  accountId: string    # Номер счёта (20 цифр)
  currency: string     # Валюта счёта
